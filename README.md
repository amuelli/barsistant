# Barsistant (Next.js + Deno)

This project is set up with Next.js at the repository root and uses Deno tasks
to run it.

## Setup

```bash
deno install --allow-scripts=npm:sharp
deno install
```

Use this to run Next.js and Convex together for local development:

```bash
deno task dev:full
```

If you need to run Convex separately, use:

```bash
deno task convex:dev
```

Use this once after adding/updating Convex functions to refresh generated types:

```bash
deno task convex:codegen
```

`deno task convex:dev` will create the `convex/` directory and update
`.env.local` with `NEXT_PUBLIC_CONVEX_URL` and `CONVEX_DEPLOYMENT`.

## Runtime Config

Required runtime variable:

- `NEXT_PUBLIC_CONVEX_URL`: Convex deployment URL used by both browser and
  server route handlers.

Optional helper variable:

- `CONVEX_DEPLOYMENT`: Convex deployment name used by Convex tooling.

Local development source of truth is `.env.local` generated by
`deno task convex:dev`.

Deno Deploy must define `NEXT_PUBLIC_CONVEX_URL` in project environment
variables so `/api/imports` and `/api/imports/[jobId]` can reach Convex at
runtime.

## Run

```bash
deno task dev
```

Open `http://localhost:3000`.

## Build and Start

```bash
deno task build
deno task start
```

Health check endpoint:

```bash
curl http://localhost:3000/api/health
```

Startup smoke check (expects built app, then verifies `/api/health`, `/`
including the `barsistant-shell-v1` app-shell marker, and import endpoints:
`POST /api/imports` returns `202 queued` and `GET /api/imports/[jobId]` returns
the persisted queued job when `NEXT_PUBLIC_CONVEX_URL` is configured; otherwise
`POST /api/imports` returns a controlled `503 import_service_unavailable`):

```bash
deno task smoke:health
```

## Lint and Check

```bash
deno task fmt
deno task lint
deno task test
deno task typecheck
deno task smoke:health
deno task check
```
